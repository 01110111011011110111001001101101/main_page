# Οδηγίες επεξεργασίας προσφορών

Οι κάρτες προσφορών εμφανίζονται δυναμικά από το `assets/data/offers.json`.

**Για να προσθέσεις νέα προσφορά, επεξεργάζεσαι μόνο το `assets/data/offers.json`.**

## Πώς προσθέτεις νέα προσφορά

1. Άνοιξε το `assets/data/offers.json`.
2. Αντέγραψε ένα υπάρχον offer object μέσα στον πίνακα `offers`.
3. Άλλαξε το `id` ώστε να είναι μοναδικό, με λατινικούς χαρακτήρες και παύλες.
4. Άλλαξε το `title` με το όνομα της προσφοράς.
5. Άλλαξε το `category`.
   Επιτρεπτές κατηγορίες: `mobile`, `internet`, `tv`, `guide`, `other`.
6. Άλλαξε το `badge`.
7. Άλλαξε το `price`, το `period` και το `monthly` όπου υπάρχει.
8. Άλλαξε το `benefits` με τα 3 βασικά bullets που εμφανίζονται στην κάρτα.
9. Άλλαξε το `brandSkin` και, αν χρειάζεται, τα σχετικά `style.cardClasses`.
10. Ρύθμισε το `ctaType`, το `modalId` ή το `actionTarget` ανάλογα με το τι πρέπει να ανοίγει το κουμπί.
11. Συμπλήρωσε τα `details`, `includes`, `appliesTo`, `documents` και `notes` για το modal “Περισσότερα”.
12. Βάλε `active: true` για να εμφανίζεται η προσφορά.
13. Τρέξε `npm run validate:offers`.
14. Τρέξε `npm run check`.
15. Κάνε commit και push.

## Πολύ σημαντικό

- Το JSON δεν υποστηρίζει σχόλια. Μη βάλεις σχόλια μέσα στο `offers.json`.
- Πρόσεχε τα κόμματα ανάμεσα στα πεδία και στα objects.
- Μην αφήνεις κόμμα μετά το τελευταίο πεδίο ενός object ή μετά το τελευταίο item ενός array.
- Μην αλλάζεις πραγματικά τηλέφωνα, emails, IBANs ή links εγγράφων χωρίς επιβεβαίωση.
- Αν προσθέσεις έγγραφο, βεβαιώσου ότι το αρχείο υπάρχει μέσα στο `assets/docs/`.

## Έλεγχος

Μετά από κάθε αλλαγή τρέξε:

```bash
npm run validate:offers
npm run check
```

Ο έλεγχος επιβεβαιώνει ότι:

- το `offers.json` είναι έγκυρο JSON,
- κάθε προσφορά έχει μοναδικό `id`,
- υπάρχουν τα βασικά απαιτούμενα πεδία,
- το `benefits` είναι array,
- το `category` είναι έγκυρη κατηγορία,
- το `active`, αν υπάρχει, είναι boolean.
